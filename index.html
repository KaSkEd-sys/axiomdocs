üìã Overview

KaskedOS is a lightweight, educational operating system developed entirely in x86 assembly language. Designed to run in real mode, it provides a simple command-line interface with essential system utilities and component management capabilities.

Key Highlights

  • Pure Assembly - Written entirely in x86 assembly (NASM)
  • Real Mode - 16-bit x86 architecture
  • Minimal Footprint - Fits in just a few kilobytes
  • Component Registry - Dynamic OS component management
  • Hidden Components - The OS includes features that are disabled by default and can be enabled through commands

Core Features

  • Custom Shell - Interactive command-line interface
  • System Information - Display OS details and current time
  • Power Management - Reboot and shutdown capabilities
  • Component Registry - Enable/disable OS features dynamically
  • Network Diagnostics - Basic ping functionality

üöÄ Getting Started

Prerequisites

Before building KaskedOS, ensure you have the following installed:

# Update package list
sudo apt update

# Install required packages
sudo apt install -y nasm qemu-system-x86

Required packages:

  • nasm - Netwide Assembler (for building)
  • qemu-system-x86 - x86 emulator (for running)

Building from Source

1. Clone the repository

git clone https://github.com/KaSkEd-sys/kaskeDOS.git
cd kaskeDOS

2. Make the build script executable

chmod +x build.sh

3. Build and run

./build.sh

The build script will:

  • Assemble the bootloader (boot.asm)
  • Assemble the kernel (kernel.asm)
  • Create a bootable disk image (kaskedos.img)
  • Launch QEMU automatically

Manual Launch

To run the OS manually after building:

qemu-system-i386 disk/kaskedos.img

First Boot

After building, KaskedOS will launch automatically in QEMU. You'll be greeted with:

=== KaskedOS v1.2 alpha ===
type help for command list

[fastuser]$kaskedos$ >

Component Registry (opreg)

The opreg utility allows you to manage OS components dynamically. Currently supports the fastfetch component.

Usage:

opreg          # Display help and available arguments
opreg -r       # Read component status
opreg -c       # Enable component
opreg -d       # Disable component

Example workflow:

[fastuser]$kaskedos$ > opreg -r
OS Components Registry:
fastfetch: DISABLED

[fastuser]$kaskedos$ > opreg -c
Component enabled successfully

[fastuser]$kaskedos$ > opreg -r
OS Components Registry:
fastfetch: ENABLED

[fastuser]$kaskedos$ > fastfetch

Hidden Features

KaskedOS includes components that are disabled by default. These features can be discovered and enabled through the component registry system using the opreg command. Use opreg -r to see which components are available and opreg -c to enable them!

üèóÔ∏è Technical Details

Architecture Specifications

  • CPU Mode: Real Mode (16-bit)
  • Assembler: NASM (Netwide Assembler)
  • Boot Protocol: Legacy BIOS
  • Disk Image: Raw binary format

Memory Layout

  • 0x7C00 - Bootloader entry point (loaded by BIOS)
  • 0x1000 - Kernel load address
  • 0x9000 - Stack pointer initialization

Boot Process

  1. BIOS loads the first sector (512 bytes) to memory address 0x7C00
  2. Bootloader initializes segments (DS, ES, SS) and stack pointer
  3. Bootloader loads 8 sectors (kernel) from disk to 0x1000 using INT 0x13
  4. Control jumps to kernel entry point at 0x1000
  5. Kernel initializes, clears screen, and starts the command shell

Key Components

Bootloader (boot.asm)

  • Sets up segment registers and stack
  • Displays loading message
  • Loads kernel sectors from disk
  • Transfers control to kernel

Kernel (kernel.asm)

  • Command parser and dispatcher
  • String comparison utilities
  • Screen management (clear, print)
  • BIOS interrupt handlers (keyboard, display, time)
  • Component registry system
  • Power management (reboot, shutdown)

BIOS Interrupts Used

Interrupt Function Usage
INT 0x10 Video services Screen clear, character output, cursor positioning
INT 0x13 Disk services Loading kernel from disk
INT 0x16 Keyboard services Reading user input
INT 0x1A Real-time clock Getting system time

ü§ù Contributing

Contributions are welcome! Whether it's bug fixes, new features, or documentation improvements.

How to Contribute

  1. Fork the repository
  2. Create a feature branch (git checkout -b feature/amazing-feature)
  3. Commit your changes (git commit -m 'Add amazing feature')
  4. Push to the branch (git push origin feature/amazing-feature)
  5. Open a Pull Request

Development Guidelines

  • Follow existing code style and conventions
  • Test thoroughly in QEMU before submitting
  • Keep commits atomic and well-described
  • Update documentation as needed
  • Comment complex assembly routines

Known Issues

  • Time display uses BIOS RTC (may not reflect actual time in emulators)
  • Shutdown command works on QEMU but may not work on real hardware
  • Limited to 64-character command input
  • No filesystem support (all data is in-memory)
